# iron-core-cluster-v2 协作硬性标准

## 依赖管理

- `Cargo.toml` 里添加独立第三方依赖，不使用 workspace 依赖。

## 注释规范

- Rust 代码统一使用 `//` 注释，不使用 `///` 或 `//!`。
- 每个数据结构上方、每个字段后方、每个方法上方都必须有中文注释。

## 变更确认

- 涉及数据结构或方法变更时，Codex/AI 必须先用表格说明变更了数据结构和方法，用户确认后再编码。

## OpenRaft 示例迁移方式

- `openraft-examples/raft-kv-memstore/` 只作为 OpenRaft 示例参考代码，迁移时必须人工阅读和理解示例，不允许机械照搬到 `src`。
- 新功能必须逐步迁移到 `src/` 下，并重新组织为适合 `iron-core-cluster-v2` 的代码结构。
- 每次迁移只推进一个清晰的小步骤，优先保证用户能学习和理解当前步骤。
- 每一步开始前必须说明本步参考了示例中的哪些文件、准备迁移什么能力、会新增或调整哪些数据结构和方法。
- 每一步完成后必须说明代码放在 `src/` 的哪个位置、它对应 OpenRaft 示例中的哪部分能力、下一步建议迁移什么。
- OpenRaft 示例源码本身原则上不修改；需要本地运行或依赖解析调整时，只允许修改示例自己的配置文件，并说明原因。
